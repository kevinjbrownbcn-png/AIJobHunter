1. System Architecture & Component Mapping

This repository houses a unified, zero-dependency standalone technical screening and document synthesis portal. The application decouples interface logic from heavy processing tasks by handing execution off to external cloud-native API routes. This ensures that UI render threads remain unblocked during high-density operations.

src/index.html UI/DOM Layout Application core interface frame. Renders execution dashboards, target job description audit fields, asset tracking sheets, and skill gap visualization components.

2. Data Lifecycle & Communication Traces

The lifecycle of an analyzed job posting follows a deterministic state machine model across three distinct transaction phases:

Phase A: Initialization & Base Asset Read

On browser boot, index.html initiates an asynchronous XMLHttp / Fetch request to the Pipeline 02 Endpoint. The endpoint performs a synchronous fetch of the base plain-text profile data and feeds it back to the local context window without triggering CORS or authentication blocks.

Phase B: Local Analysis & Export Schema Mapping

When a job description is pasted into the manual workspace, the application extracts its contents and uses the internal configuration context to evaluate alignment. Once validated, clicking the export control triggers exporter.js to structure a standardized JSON payload packet:

{
  "profile": "Localization" | "Data Analysis" | "Tech Support",
  "job_title": "String (Dynamic)",
  "company": "String (Dynamic)",
  "link": "URL_String",
  "description": "Text_Block",
  "match_score": "Integer",
  "skills_gaps": "Markdown_String",
  "resources": "Markdown_String",
  "status": "Draft"
}

This packet is instantly transmitted to the Pipeline 03 Webhook to populate the triage spreadsheet row array.

Phase C: Cloud Document Synthesis Route

When a row's status state transitions from Draft to New, Pipeline 01 activates. The conditional routing engine evaluates the profile value and branches the execution path:

  • Localization Operations Track: Compiles content utilizing targeted language technology, validation systems, and vendor program delivery matrices.
  • Data Analysis Track: Strips translation jargon. Re-frames work arrays to stress Python data parsing, spreadsheet data integrity modeling, pipeline automation, and Regex string sanitization. Inject a career-pivot statement detailing systems analytics experience.
  • Tech Support / IT Track: Translates project management experience into service request routing, error triage, software environment diagnostics, deployment configurations, and incident escalation controls.

To run, modify, or version control this workspace locally, adhere to the following directory layout protocol:

How to Implement Changes Safely

  1. Modifying the UI Layout: Open src/index.html. When updating DOM elements or script references, ensure that the element IDs match the expected query selectors used in the inline event handling routines.
  2. Modifying the Export Logic: Open src/exporter.js. If you append additional columns to your Google Sheets triage grid, you must update the payload construction model within the export loop array, then map the corresponding keys inside Module 2 of Pipeline 03 on Make.com.
  3. Staging & Committing to Git: Run the following standard commands to track file state modifications cleanly: 
    git status
git add src/index.html src/exporter.js
git commit -m "feat: implement multi-profile branching pipeline logic"
git push origin main

4. Emergency Runbook & Troubleshooting Routines

Symptom 1: Export Actions Fail / Webhook Responds with Error Alerts

  • Root Cause Analysis: The target Make.com webhook endpoints have either changed their tracking hash IDs, or the global scheduling toggles inside the Make interface have been switched to an inactive state.
  • Resolution Protocol: Open Pipeline 03 inside Make. Verify that the system shows an active scheduling state. Copy the webhook URL and ensure it matches the target destination parameter configured within your application's transmission script.

Symptom 2: Generated Resumes Overrun Single-Page Margins

  • Root Cause Analysis: The AI completion engine generated long paragraphs or redundant sentences that caused text to overflow.
  • Resolution Protocol: Open Pipeline 01. Locate the specific user prompt for the affected branch path. Tighten the constraint language under the SPATIAL EFFICIENCY directive, explicitely ordering the model to generate tighter, high-density bullet arrays.

Symptom 3: Multi-Branch Routing Fails to Execute

  • Root Cause Analysis: String casing mismatches between the spreadsheet profile cells and the Make router logic filters.
  • Resolution Protocol: Check Column A in your spreadsheet tracker. Ensure strings like Data Analysis match the Make router logic configuration exactly. Avoid introducing extra trailing spaces or variations in text capitalization.
